Cedric Herreman wrote:
> Can we use USB4RT to read images from a camera ? We

In principle yes, but it strongly depends on the camera. Real-time or
not, you need documentation or - even better - a high-level USB
reference driver for the particular device.

> are using Linux with RTAI patch.

Note that RT4USB is yet limited to RTAI/fusion. The plan for the future
is to build RT4USB on top of RTDM, which will hopefully soon be fully
supported by RTAI 3.x (more precisely: from upcoming 3.3 onwards) as well.

> 
> Our PC-board only has USB 1.1, we think this will be
> too slow to read the images fast enough. We are
> thinking about mounting a PC/104 extension board that
> supports USB 2.0. Does this pose a problem for the
> USB4RT ?

Not a problem except that, even on USB 2.0 hardware, RT4USB is so far
limited to 1.1 speed. EHCI support for faster communication is scheduled
for future development.

> 
> Is Firewire a better alternative than USB 2.0 for
> reading camera pictures ?

Can be better if the documentation of the camera interface is more
complete. Firewire cameras typically comply to open standard profiles
while USB webcam etc. tend to work with proprietary interfaces. Of
course, both domains contain exceptions.
