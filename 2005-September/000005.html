Cedric Herreman wrote:
> I read in the README file that usb4rt requires a 2.6
> kernel and Fusion 0.8.0 or later. Are these hard
> requirements ? 

For now: yes. More recent Fusion versions should work as well or will
only require minor changes.

> 
> Can it be ported to a 2.4 kernel ? Can a RTAI version
> 3.0 be used ?
> 

It can, but it takes always someone to work on this.

For us, everything below RTAI 3.3 is not on the roadmap as those
versions will likely never support the abstraction layer RTDM. Even for
magma (i.e. upcoming 3.3), RTDM is not yet working, but it should do so
soon (according to the maintainters).

As stated before, USB4RT is planned to be ported over RTDM which will
overcome the need to have its own abstraction of the RTOS version below.
