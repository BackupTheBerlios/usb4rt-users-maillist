Cedric Herreman wrote:
> Hi.
> 
> I managed to compile & install xenomai 2.0 and usb4rt
> on a target system. But I cannot usb-ping a device. I
> checked the usb_dev list : it is empty.
> 
> I found that the host controller(s) enter(s) the error
> state : "Host System Error". The UHCI-spec states that
> this can be caused by PCI communication problems.
> 
> I have noticed that after using the rt4usb modules, my
> network is not working correctly anymore. I have to
> "/sbin/ifdown eth0" and "/sbin/ifup eth0" to get it
> working again.
> 
> Both Ethernet controllers and USB controllers are on
> the PCI bus : can this be the cause of problems ? 

I would rather guess: IRQ conflict. Please check first if the USB and
the Ethernet controllers are sharing the same IRQ. Try to avoid this,
e.g. by plugging the NIC in some other PCI slot - as long as it's not
on-board. In the latter case, try playing with the BIOS, maybe even
disable the NIC for a simple test.

Jan
