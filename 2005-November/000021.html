I have just found the problem : apperently the host
controller (VIA) on my hardware has different
behavior.

In function uhc_init (rt_uhci.c) the framelist-address
is written to the register together with some other
configurations (framenumber, max packet size for
reclamation).

In function uhc_start that follows a host controller
reset is triggered. On my hardware this causes the
framelist-address register to be cleared ! The other
configurations in the registers remain unchanged. If
the host controller is started it will read address
NULL : this causes the Host System Error.

If the framelist-address register is rewritten after
the reset, all works ok. 

I can now ping the device.

Cedric.
